# Physical AI & Humanoid Robotics Textbook Constitution

## Core Principles

### I. Prime Directives (The "Must-Haves")
* AI-Generated Content Only: All content (text, code examples, structure) must be generated by AI (Claude/Gemini). Manual intervention is allowed only for file structure, navigation fixes, or critical corrections.
* Submission-First Strategy: The primary objective is a deployed Docusaurus website with high-quality chapters. All other features (RAG Chatbot, Auth, Translation) are secondary and must not block the publication of the book.
* Physical AI Focus: The content must strictly adhere to the "Physical AI & Humanoid Robotics" curriculum (ROS 2, Gazebo, NVIDIA Isaac, VLA). No generic AI content.

### II. Writing Standards (Style & Tone)
* Target Audience: Grade 9-12 students and beginners.
* Tone: Educational, encouraging, yet technically accurate. "Serious fun."
* Clarity: Use simple English. Explain jargon (e.g., "Inverse Kinematics", "Sim-to-Real") immediately upon use.
* Structure:
    * Short paragraphs (3-4 sentences max).
    * Frequent use of bullet points for readability.
* Micro-Summaries: Every major section must end with a 1-sentence takeaway.

### III. Formatting & Technical Rules
* Platform: Docusaurus (Markdown/MDX).
* File Structure:
    * All chapters live in /docs.
    * Folder naming: kebab-case (e.g., physical-ai-intro).
    * Images: Store in /static/img, reference via relative paths.
* Code Blocks: All code examples (Python, C++, XML/URDF) must use proper syntax highlighting (e.g., ```python).
* Visuals:
    * Use `` tags to instruct the Image Generation tool (nano_banana or similar) where to place diagrams.
    * Do not leave empty placeholders without instructions.

### IV. Content Quality Control
* Word Count: Target 400-600 words per sub-chapter (digestible chunks).
* Fact-Check: ROS 2 commands must be compatible with "Humble" or "Iron" distributions.
* Hardware Awareness: Explicitly mention hardware constraints (e.g., "This requires an NVIDIA GPU") where necessary, as per the hackathon guide.

### V. Architecture & workflow
* Phase 1 (Critical): Generate Chapters 1-4 → Build Docusaurus → Deploy to GitHub Pages.
* Phase 2 (Bonus): Implement RAG Chatbot (FastAPI/Qdrant).
* Phase 3 (Bonus): Add Auth & Personalization buttons.

## Governance
This Constitution governs the foundational principles and operational guidelines for the 'Physical AI & Humanoid Robotics Textbook' project. Amendments require documentation, approval by the project lead, and a clear migration plan for any affected processes or artifacts. Compliance reviews are conducted quarterly.
