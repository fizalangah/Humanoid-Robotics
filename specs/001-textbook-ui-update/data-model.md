# Data Model for AI Textbook & Modern UI

This data model outlines the key entities involved in the AI Textbook & Modern UI feature, based on the feature specification. Given this is primarily a static content generation and display application built with Docusaurus, the data model is conceptual and focuses on the structure of information rather than a persistent database schema.

## Entities

### 1. Chapter

Represents a single educational chapter within the textbook.

**Attributes**:
-   **Title**: The title of the chapter (e.g., "Introduction to Physical AI").
-   **ID**: A unique identifier for the chapter (e.g., "001").
-   **Route**: The URL path for accessing the chapter (e.g., `/docs/physical-ai/introduction`).
-   **Content**: The main educational text, likely in MDX format, including paragraphs, bullet points, and code blocks.
-   **Word Count**: The target length of the chapter content (400-600 words per sub-chapter).
-   **Micro Summary**: A single-sentence takeaway at the end of major sections.
-   **Dependencies**: Other chapters that must be read or understood before this one (e.g., Chapter 1).
-   **Visuals**: References to images embedded within the chapter.
-   **Hands-on Steps**: (Optional) Specific interactive or practical steps included in some chapters.

**Relationships**:
-   Has many `Images`.

### 2. Image

Represents a visual asset used within the textbook, typically generated by the `nano_banana` tool.

**Attributes**:
-   **Source**: The file path or URL to the image (e.g., `/static/img/hero-robot-hand.png`).
-   **Alt Text**: Descriptive alternative text for the image.
-   **Prompt**: The descriptive prompt used to generate the image via `nano_banana`.
-   **Placement Context**: Where the image is displayed (e.g., "Hero Section", "Chapter 2 - Humanoid Anatomy").

**Relationships**:
-   Belongs to one `Chapter` or the Landing Page.

### 3. User

Represents an individual interacting with the textbook website.

**Attributes**:
-   (No specific attributes are defined or required for this feature, as there's no authentication or personalization in scope for the initial release).

**Relationships**:
-   Interacts with `Chapters` and the Landing Page.

## Considerations

-   This model is primarily for content organization and generation. It does not imply a relational database or complex data persistence layer for the initial release.
-   The content and images will be managed as static files within the Docusaurus project structure.
