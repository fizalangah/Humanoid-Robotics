## User Scenarios & Testing *(mandatory)*

### User Story 1 - Navigate Repaired Curriculum (Priority: P1)

As a learner, I want to access all 9 chapters of the curriculum through a correctly functioning sidebar, without encountering any "Page Not Found" errors, so that I can have a seamless learning experience.

**Why this priority**: This is a critical fix to the core functionality of the textbook website.

**Independent Test**: Verify that each of the 9 chapter links in the sidebar correctly navigates to its corresponding page.

**Acceptance Scenarios**:
1.  **Given** I am on the textbook website, **When** I view the sidebar, **Then** I see 9 distinct chapter entries, ordered numerically.
2.  **Given** I click on any of the 9 chapter entries in the sidebar, **When** the page loads, **Then** the content for that chapter is displayed correctly.

## Requirements *(mandatory)*

### Functional Requirements

-   **FR-001**: The documentation structure MUST be reverted to a nested folder structure.
-   **FR-002**: Every chapter MUST have its own folder inside `docs/`.
-   **FR-003**: Inside each chapter folder, the chapter content file MUST be named `index.mdx`.
-   **FR-004**: Each `index.mdx` file MUST have a `title` in its frontmatter.
-   **FR-005**: The `sidebars.ts` file MUST be configured to use `autogenerated` for the `docs` directory.
-   **FR-006**: Each chapter page MUST display the specified Unsplash image.
-   **FR-007**: All links in the sidebar and on the homepage MUST correctly navigate to their corresponding chapter pages.

## Success Criteria *(mandatory)*

### Measurable Outcomes

-   **SC-001**: The `npm run build` command completes successfully with no broken link errors.
-   **SC-002**: All 9 chapters are accessible from the sidebar and display their content correctly.
-   **SC-003**: No "Page Not Found" (404) errors are encountered when navigating the site.
